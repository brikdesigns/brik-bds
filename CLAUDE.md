> **Cross-repo rules + canonical token registry + brik-secrets workflow** load via walk-up from `~/Documents/Github/CLAUDE.md`. USE `brik-secrets <subcommand>` for any credential operation; never write a custom rotator.

## STOP — Canonical Token Registry (Non-Negotiable)

Canonical names: [design.brikdesigns.com/docs/primitives/color](https://design.brikdesigns.com/docs/primitives/color), mirrored in `dist/tokens.css` (live allowlist).

EVERY `--text-*` / `--surface-*` / `--background-*` / semantic `--border-*` / `--color-*` token name in any Brik repo MUST exist in this registry. → rag:tokens

NEVER invent:
- `--text-on-{surface}` beyond `--text-on-color-dark` / `--text-on-color-light`
- `--surface-paper` / `--surface-ink` / `--surface-accent-*`
- Tier modifiers (`-soft`, `-link`, `-hover`) on `text-on-*` names

USE the existing semantics: `--text-primary` / `--text-secondary` / `--text-muted`; `--surface-primary` / `--surface-secondary` / `--surface-inverse` / `--surface-overlay`.

ADD new semantic tokens via Figma → `tokens-studio.json` → `npm run build:sd-figma`. NEVER ship inline aliases as a "temporary fix" — `scripts/token-audit.sh --canonical-only` enforces.

## STOP — Token Rules (Non-Negotiable)

NEVER manually edit `tokens/figma-tokens.css` or `tokens/figma-tokens-dark.css` — both auto-generated by Style Dictionary.

Consumer projects IMPORT `figma-tokens-dark.css`; never write `[data-theme="dark"]` blocks.

To pull tokens from Figma — dev plugin + WebSocket relay only:

```bash
bun scripts/pull-variables.js <channel-id> > /tmp/figma-vars.json
node scripts/sync-figma-mcp.js /tmp/figma-vars.json --build
```

NEVER use Figma REST API (Enterprise-only), `mcp__claude_ai_Figma__get_variable_defs`, or `mcp__figma-desktop__get_variable_defs` — none give file-level token data.

VERIFY any `var(--...)` reference exists in `tokens/figma-tokens.css` or `tokens/gap-fills.css` first. READ [`tokens/CASCADE.md`](tokens/CASCADE.md) before editing any `tokens/*.css` file.

## Token Discipline (ALL consuming projects)

1. **Three-layer architecture:** BDS foundations (`figma-tokens.css`) → interaction states (`gap-fills.css`) → client domain (`theme-{client}.css`). NEVER skip layers.
2. **Stay in semantic category:** `--text-*` for text, `--background-*` for backgrounds, `--border-*` for borders. NEVER cross-use.
3. **Color = token reference.** No hex, `filter` hacks, or `opacity` workarounds.
4. **Before rewriting `:root`** or any token file, GREP `var(--...)` usages — a removed token silently breaks every reference.
5. **Background change → contrast check** on paired text/icon tokens.
6. **NEVER guess values.** Pull from Figma data or ask.
7. **NEVER hand-write dark mode.** Consumers import `figma-tokens-dark.css`.
8. **Single-source components.** NEVER duplicate forms/views/interactives across routes.
9. **Storybook MCP first.** Query `list-all-documentation` / `get-documentation` before building custom UI.
10. **Brik internal service tokens stay internal.** `--background-service-*`, `--text-service-*`, `--services--yellow-light` MUST NOT appear in client-product token files.

## Paper — pre-implementation design

USE Paper for new/composite components before writing code — renders real HTML/CSS instantly. Only BDS token names (`var(--text-primary)`, `var(--padding-lg)`); never raw values.

SKIP Paper for simple variants of existing components or when a clear Figma spec exists. → [`docs/COMPONENT-PATTERNS.md`](docs/COMPONENT-PATTERNS.md) "Pre-implementation: Design in Paper".

## Radix UI primitives

USE Radix UI for complex interactive components needing a11y foundations (focus trapping, keyboard nav, ARIA, portal rendering). Radix handles behavior; BDS owns styling via CSS custom properties.

INSTALL one `@radix-ui/react-*` per component need (today: `@radix-ui/react-popover` for DatePicker). NEVER use shadcn/ui or component-library CSS variables.

## Token System

USE Figma single-dash names: `--text-primary`, `--background-brand-primary`, `--padding-lg`.

NEVER use the legacy Webflow double-underscore prefix (`--_color---text--primary`) in new code — it exists in `tokens/variables.css` for the Webflow platform only.

Build pipeline: Figma Variables → Tokens Studio JSON → Style Dictionary → per-platform outputs.

Canonical registry: [Primitives](https://design.brikdesigns.com/docs/primitives). Rename history: [`docs/TOKEN-RENAMES.md`](docs/TOKEN-RENAMES.md).

NEVER load fonts in BDS — consuming projects load their own. `--font-family-*` tokens reference families by name.

### Spacing modes

Three modes (`compact` / `comfortable` / `spacious`) modulate `--padding-*` and `--gap-*` via `[data-mode-spacing="..."]` on `:root`. Default mode requires no attribute.

Auto-generated values: [`tokens/modes-spacing.css`](tokens/modes-spacing.css). Storybook overrides via `storybook-overrides.css`.

## Content System (BCS)

BCS is the content-vocabulary peer to the token system at `content-system/`. Separate npm entry: `@brikdesigns/bds/content-system`.

Architecture + authoring guide: [`content-system/README.md`](content-system/README.md).

### BCS Discipline

1. **Locked enums = single source.** `PERSONALITY_VALUES`, `VOICE_VALUES`, `VISUAL_STYLE_VALUES`, `INDUSTRY_SLUGS` exported from BCS. NEVER hardcode in consumer code.
2. **Industry slug registry + packs in sync.** New `INDUSTRY_SLUGS` entry without matching `industries/{slug}.ts` + `.mdx` fails typecheck. NEVER bypass with `as`.
3. **Versioning + review cadence mandatory.** `version`, `reviewCadence`, `lastReviewed` per pack. BUMP on content change (semver); UPDATE `lastReviewed` post-review.
4. **The catch-all graduates.** `small-business` is `DEFAULT_INDUSTRY_SLUG`. GRADUATE when 3+ clients OR seasonality/regulation/terminology diverges OR strategy docs repeat 60%+.
5. **Voice patterns compose, not lookup.** Each voice defines structured rules; portal resolver blends 3 picks at 60/30/10 weighting.
6. **Client overrides win over industry defaults.** `company_profile.cta_language` / `anti_messages` / `naming_conventions` override the industry pack.
7. **NEVER use literal `<` in MDX copy.** MDX v3 parses `<` as JSX; USE "under", `&lt;`, or backticks.
