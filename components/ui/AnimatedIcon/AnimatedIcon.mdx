# Animated icon

<ComponentLinks slug="animated-icon" />

Lottie-react wrapper for animated UI icon states. Consumers source Lottie JSON from [useanimations.com](https://useanimations.com), [lottiefiles.com](https://lottiefiles.com), or their own After Effects → Bodymovin exports and pass it via `animationData`.

## Variants

AnimatedIcon has no semantic-variant axis — every prop (`trigger`, `size`, `loop`, `label`) is a Control on the canvas above. Per [ADR-010 §components without a variant axis](https://github.com/brikdesigns/brik-bds/blob/main/docs/adrs/ADR-010-storybook-axes-of-information.md), the `trigger` enum (`loop` / `hover` / `click` / `once`) lives in Controls rather than as separate stories because at-rest snapshots are visually identical across modes — trigger differences only manifest in motion.

<Canvas of={Stories.Default} />

- **`loop`** — continuous decorative motion
- **`hover`** — discoverable interaction hint
- **`click`** — feedback on activation
- **`once`** — entrance moment on mount

## Notes

> **Note** — `lottie-react` respects `prefers-reduced-motion` when `autoplay` is `false`. AnimatedIcon sets `autoplay` only for the `loop` and `once` triggers; `hover` and `click` are user-initiated, so motion-sensitive users hit them deliberately.

> **Note** — BDS does not ship a curated Lottie catalog. Each consumer brings its own animation JSON files. If you need a system-wide catalog (similar to the iconify-backed `Icons` reference), file a separate issue.
