# Button

Triggers an event or action. The button family includes three components for different semantic needs.

## Variants

Nine variants split into brand (primary/outline/secondary/ghost/inverse/on-color) and system (selected/destructive/positive).

<Canvas of={Stories.Variants} />

- **`primary`** — Main call-to-action. One per section.
- **`outline`** — Secondary emphasis. Alternative to primary.
- **`secondary`** — Tertiary/subtle. Supporting actions.
- **`ghost`** — Minimal emphasis. Navigation, repeated actions.
- **`inverse`** — High-contrast for dark surfaces.
- **`on-color`** — High-contrast for branded surfaces.
- **`destructive`** — Destructive primary action (delete, remove).
- **`positive`** — Affirmative semantic action.
- **`selected`** — Active/selected toggle state.

## LinkButton

An `<a>` element styled as a button. Use when the action navigates to a URL. The `href` prop is required.

<Canvas of={Stories.LinkButtonShowcase} />

```tsx
import { LinkButton } from '@brikdesigns/bds';

<LinkButton href="/docs" variant="outline">Read docs</LinkButton>
```

## IconButton

An icon-only button with a required `label` prop for accessibility.

<Canvas of={Stories.IconButtonShowcase} />

```tsx
import { IconButton } from '@brikdesigns/bds';

<IconButton icon={<CloseIcon />} label="Close dialog" variant="ghost" />
<IconButton icon={<TrashIcon />} label="Delete item" variant="destructive" />
```

## Props

### Button

<ArgTypes of={Stories} />

### LinkButton

- **`href`** `string` *required* — Link destination.
- **`variant`** `ButtonVariant` — Visual style variant. Default `'primary'`.
- **`size`** `ButtonSize` — Button size. Default `'md'`.
- **`fullWidth`** `boolean` — Full width. Default `false`.
- **`iconBefore`** `ReactNode` — Icon before text.
- **`iconAfter`** `ReactNode` — Icon after text.

### IconButton

- **`icon`** `ReactNode` *required* — The icon to render.
- **`label`** `string` *required* — Accessible label (`aria-label`).
- **`variant`** `ButtonVariant` — Visual style variant. Default `'ghost'`.
- **`size`** `ButtonSize` — Button size. Default `'md'`.
- **`loading`** `boolean` — Loading state. Default `false`.

---

## Usage

```tsx
import { Button, LinkButton, IconButton } from '@brik/bds';

// Standard action
<Button variant="primary" size="md">Save changes</Button>

// Navigation
<LinkButton href="/docs" variant="outline">Documentation</LinkButton>

// Icon-only
<IconButton icon={<EditIcon />} label="Edit" variant="secondary" />

// Destructive action
<Button variant="danger">Delete project</Button>
```

## CSS Override API

Component-scoped CSS variables exposed on `.bds-button`. Set these on a wrapping element or directly on the button to override without touching BDS internals.

| Variable | Default | Controls |
|---|---|---|
| `--button-active-translate-y` | `1px` | Downward shift on `:active` press |
| `--button-focus-outline-width` | `2px` | Focus ring stroke width |
| `--button-focus-outline-offset` | `2px` | Gap between button edge and focus ring |

```css
/* Example: flatten press effect in a toolbar context */
.my-toolbar {
  --button-active-translate-y: 0;
  --button-focus-outline-width: 3px;
}
```
