# Sheet

<ComponentLinks slug="sheet" />

Sliding panel overlay anchored to a screen edge. Use for detail views, forms, and secondary content. Two modes — `read` (view) and `edit` (form) — drive the auto-footer (`[Close] [Edit]` vs `[Cancel] [Save]`).

## Playground

<Canvas of={Stories.Playground} />

## Variants

Right, left, bottom, wide, and no-title configurations.

<Canvas of={Stories.Variants} />

- **`right`** — Default, slides from right edge
- **`left`** — Navigation or sidebar panels
- **`bottom`** — Mobile-friendly bottom sheet

### Header

The header composes an optional `subtitle` eyebrow, the `title`, an optional `description`, and an optional `onBack` button for nested flows.

<Canvas of={Stories.Header} />

### Read mode

Read mode displays data. When `onEdit` is supplied, the footer auto-renders `[Close] [Edit]`.

<Canvas of={Stories.ReadMode} />

### Edit mode

Edit mode is the active form state. The footer auto-renders `[Cancel] [Save]`. Pass `saveLoading` while the save promise is in flight and `saveDisabled` when the form is invalid.

<Canvas of={Stories.EditMode} />

### Read ↔ Edit toggle

The canonical pattern: open in `read`, click Edit to switch to `edit`, Save / Cancel returns to `read`.

<Canvas of={Stories.ReadEditToggle} />

### Tabs

Tabs render below the header. Each tab renders its own body content — `children` is ignored when `tabs` is supplied.

<Canvas of={Stories.Tabs} />

### Secondary action

For ancillary actions next to the primary Edit / Save — e.g. *Refresh*, *Re-generate* — pass `secondaryAction` instead of composing a custom `footer`. Renders left-aligned; mode-driven primary actions stay on the right. Suppressed in edit mode.

<Canvas of={Stories.SecondaryAction} />

### Read-only floating

Pure read content with no action — `variant="floating"`, no backdrop, no footer.

<Canvas of={Stories.ReadOnlyFloating} />

### Read mode floating

Read mode with `variant="floating"` — sheet sits on the page without a backdrop, footer still renders `[Close] [Edit]`. Use for inline drill-in from a table row.

<Canvas of={Stories.ReadModeFloating} />

### Tabs + Secondary action + Read/Edit

The full sheet: tabs in the header, secondary action in the footer, and read-edit toggling. Tabs are typically suppressed in edit mode so the form gets the full body.

<Canvas of={Stories.TabsWithSecondaryAction} />
