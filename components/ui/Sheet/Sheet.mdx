# Sheet

<ComponentLinks slug="sheet" />

Sliding panel overlay anchored to a screen edge. Use for detail views, forms, and secondary content. Two modes — `read` (view) and `edit` (form) — drive the auto-footer (`[Close] [View details] [Edit]` vs `[Cancel] [Save]`). `[View details]` is conditional on `viewDetailsAction`; omit it when the sheet has no corresponding read page (e.g. `plans`).

## Default

<Canvas of={Stories.Default} />

`side` (right / left / bottom), `variant`, `width`, `title` / `subtitle` / `description`, and the `closeOn*` / `showCloseButton` behaviors are all Controls on this story.

## Variants

### Read mode

Read mode displays data. When `onEdit` is supplied, the footer auto-renders `[Close] [Edit]`.

<Canvas of={Stories.ReadMode} />

### Edit mode

Edit mode is the active form state — footer auto-renders `[Cancel] [Save]`. Pass `saveLoading` while the save promise is in flight and `saveDisabled` when the form is invalid.

<Canvas of={Stories.EditMode} />

### Floating

`variant="floating"` — the panel sits on the page without a backdrop, rounded and elevated. Use for inline drill-in from a table row or a read-only popover. Omit `mode` for a footer-less info popover.

<Canvas of={Stories.Floating} />

### Compact density

`density="compact"` (orthogonal to `variant`; shared axis with Tag, Badge, BulletList, Table) tightens the body and footer vertical padding one step — body `padding-block` xl→lg, footer lg→md — for data-dense sheets. Default `comfortable` is unchanged. Horizontal padding stays put, so the body/footer keep alignment with the header.

<Canvas of={Stories.Compact} />

### Loading

`loading` swaps the header (title / subtitle / description) and body for `Skeleton` placeholders — 3 `SheetSection`-shaped blocks, each a heading line plus 4 field rows. The footer renders unchanged (the sheet stays closable); `secondaryAction` and `tabs` are suppressed while loading.

<Canvas of={Stories.Loading} />

## Patterns

### Read ↔ edit toggle

The canonical pattern: open in `read`, click Edit to switch to `edit`, Save / Cancel returns to `read`.

<Canvas of={Stories.ReadEditToggle} />

### Read footer with View details

Sheet+page hybrid tables (services, offerings, service_lines, industry_pages) surface three read-mode actions: `[Close]` · `[View details]` (navigates to the read page) · `[Edit]` (navigates to the edit page). Use `editTarget="page"` to signal that `onEdit` navigates rather than flipping to in-place edit mode.

<Canvas of={Stories.WithViewDetails} />

### Secondary action

For ancillary actions next to the primary Edit / Save — e.g. *Refresh*, *Re-generate* — pass `secondaryAction` instead of composing a custom `footer`. Renders left-aligned; mode-driven primary actions stay on the right. Suppressed in edit mode.

<Canvas of={Stories.WithSecondaryAction} />

### Tabs

Tabs render below the header. Each tab renders its own body content — `children` is ignored when `tabs` is supplied.

<Canvas of={Stories.WithTabs} />

### Custom footer

The `footer` slot fully overrides the mode-driven auto-footer — use it for bespoke action sets (e.g. a destructive `Archive` alongside `Cancel` / `Save`) that the `read` / `edit` footers don't cover.

<Canvas of={Stories.CustomFooter} />
