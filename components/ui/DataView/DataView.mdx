# DataView

<ComponentLinks slug="data-view" />

A family of four thin shells — `TableView`, `ListView`, `ProfileView`, `BoardView` — that own the loading, empty, and error states every data display needs, so consumers stop rebuilding them per page. Pass the display primitive (`Table`, `CardList`, a `DataSection` stack, a `Board`) as `children`; the view renders exactly one state in precedence order — error → loading → empty → content.
