# FieldGrid

Equal-column grid for laying out `Field` components side by side. Replaces the inline `display: grid; gridTemplateColumns: '1fr 1fr'` pattern scattered across portal sheets.

Accepts any children — `Field` is the primary case, but `Card`, `Tag`, or any equal-weight row content works too (e.g. color-primitive swatches, stat tiles).

## Columns

`columns` locks to `2`, `3`, or `4`. No custom column templates — if you need something else, you're composing the wrong primitive.

## Stat tiles

Wrapping each grid cell in a `<Card variant="outlined">` gives you the stat-tile pattern (e.g. Content Scrape's SOURCE / SCRAPED / FAILED tiles).

## Usage

```tsx
import { FieldGrid, Field, Card } from '@brikdesigns/bds';

// 2-column field grid
<FieldGrid>
  <Field label="Owner">Nick Stanerson</Field>
  <Field label="Status">Active</Field>
</FieldGrid>

// 3-column stat tiles
<FieldGrid columns={3} gap="md">
  <Card variant="outlined"><Field label="Scraped">8</Field></Card>
  <Card variant="outlined"><Field label="Failed">12</Field></Card>
  <Card variant="outlined"><Field label="Source">birdwelldentist.com</Field></Card>
</FieldGrid>
```

## Rules

- **No custom columns.** If you need 5 cells or variable widths, you're reaching for the wrong primitive — use `Table`.
- **All children must be equal-weight.** Don't mix a tall card with a single-line Field in the same grid.
