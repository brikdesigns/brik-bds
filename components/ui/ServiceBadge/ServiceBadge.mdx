# Service Badge

Colored badge identifying Brik service categories. Each category has a dedicated color palette used across the Brik ecosystem for consistent service identification.

Available in **badge mode** (colored square with optional icon) and **label mode** (pill-shaped text label).

## Usage

```tsx
import { ServiceBadge } from '@brik/bds';

// Category badge (no icon, just colored square)
<ServiceBadge category="marketing" />

// With specific service icon
<ServiceBadge category="brand" serviceName="Premium Logo Design" />

// Sizes
<ServiceBadge category="product" size="sm" />
<ServiceBadge category="product" size="lg" />

// Label mode (pill-shaped text)
<ServiceBadge category="service" mode="label" />
<ServiceBadge category="information" mode="label" size="lg" />
```

---

## Icon resolution

The `serviceName` prop resolves to an SVG icon via two mechanisms:

- **Explicit overrides** — common service names map directly to icon files (e.g., "Premium Logo Design" resolves to `brand-logo.svg`)
- **Normalization** — service names are lowercased and hyphenated, then prefixed by category (e.g., "Brand Guidelines" resolves to `brand-guidelines.svg`)

Icons are stored in `components/ui/ServiceBadge/icons/[category]/` and served from `.storybook/public/icons/` in Storybook.
