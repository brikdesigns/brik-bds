# Table

<ComponentLinks slug="table" />

Data table with composable sub-components (`TableHeader` / `TableBody` / `TableRow` / `TableHead` / `TableCell` / `TableActionsCell`, plus the typed media cells `TableAvatarCell` / `TableImageCell` / `TableServiceTagCell`). Supports sorting, row selection, striped rows, size variants, and arbitrary cell content. Trailing `[View][Edit][⋯]` clusters compose via `<TableActionsCell>`; cell-level affordances (Name / FK / Actions) replace whole-row click — see [Read and edit conventions](https://github.com/brikdesigns/brik-client-portal/blob/staging/.claude/references/settings-ia.md#read-and-edit-conventions).

## Default

<Canvas of={Stories.Default} />

`striped`, `size` (`default` / `comfortable`, 72px cell height), `headerBorder` (off by default), `headerBorderWeight` (`md` default / `sm` to match the data-row divider), `roundedTop` / `roundedBottom` (outer corners, on by default), and `headerBackground` (`secondary` / `primary`) are all Controls on this story.

## Variants

Sortable headers (`sortable` + `sortDirection` on `<TableHead>`) and checkbox row selection (`selected` on `<TableRow>`) — both are sub-component props, so they live in a dedicated story rather than Controls.

<Canvas of={Stories.SortableWithSelection} />

A `<TableCell>` accepts any content. This table exercises the composition catalog in one place — icon-left name, two-line owner, `<Tag>`, status `<Badge>`, inline `<TextInput>`, a `<Tooltip>` header indicator, and a `<TextLink>`.

<Canvas of={Stories.CellTypes} />

## Patterns

The trailing `[View][Edit][⋯]` cluster composes with `<TableActionsCell>` — it owns alignment, shrink-to-content width, and the `--gap-sm` rhythm, so consumers stop hand-rolling `style={{ textAlign: 'right' }}` on `<TableCell>`.

<Canvas of={Stories.WithActionsCell} />

The canonical read/edit row has three click-target classes — **Name** (`<TextLink>` opening the read sheet), **foreign-key cells** (`<TextLink>` opening the related entity's read sheet), and **Actions** (`<TableActionsCell>`). Read-only display cells (Status / Public / Featured) stay plain `<Badge>` or text.

<Canvas of={Stories.CellLevelInteractivity} />

### Forbidden — whole-row click

`onClick` on `<TableRow>` violates table cell semantics, breaks screen-reader expectations, and conflicts with cell-level affordances. Use cell-level click targets instead — the row should never carry the handler.

<Canvas of={Stories.RowClickAntiPattern} />

### Media cells

`<TableAvatarCell>` pairs an `Avatar` with a name and an optional secondary line (email / role). Initials render when no `src` is set; pass `primary` as a `<TextLink>` when the name should navigate.

<Canvas of={Stories.WithAvatarCell} />

`<TableImageCell>` renders a fixed square (1:1) thumbnail for a logo or product image — `fit="contain"` for logos (no crop), `fit="cover"` for product photos. Upload / edit wiring is consumer-side.

<Canvas of={Stories.WithImageCell} />

`<TableServiceTagCell>` hosts one or more `ServiceTag`s — `variant="icon"` for icon-only classification, `variant="icon-text"` for a labeled tag.

<Canvas of={Stories.WithServiceTagCell} />
