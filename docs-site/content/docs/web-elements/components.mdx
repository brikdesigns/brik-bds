The HTML/CSS surface of BDS components. While [product Components](/docs/components) are React with `onClick` handlers, Web Components are semantic HTML with BDS class names — the same `.bds-button`, `.bds-card`, `.bds-field` rules from `dist/styles.css`, but composed as static markup that Astro sites and Webflow projects can drop in directly.

## Pattern

A product Button:

```tsx
<Button variant="primary" size="md" onClick={handleClick}>Get started</Button>
```

The Web Element counterpart:

```html
<a href="/start" class="bds-button bds-button--primary bds-button--md">
  Get started
</a>
```

Same CSS. Same brand tokens. Same hover/active states. Different semantics — a website button is an `<a>` with `href`, not a JS-driven `<button>`.
