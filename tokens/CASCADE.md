# Token CSS Cascade

Canonical reference for what each file in `tokens/` does, where it loads,
and how to decide where a change belongs.

## What ships to consumers

`dist/tokens.css` is the bundle shipped to downstream consumers (portal,
renew-pms, brikdesigns) via `@brikdesigns/bds/tokens.css`. Built by
`scripts/build-dist-tokens.js`, which concatenates тАФ in this order:

1. `figma-tokens.css` тАФ auto-generated light-mode tokens (`:root`)
2. `figma-tokens-dark.css` тАФ auto-generated dark-mode tokens (`:root[data-theme="dark"]`)
3. `figma-dark-corrections.css` тАФ manual overrides for known-wrong Figma dark values
4. `theme-brand-brik.css` тАФ Brik brand overrides (scoped to `.theme-brand-brik` class)
5. `gap-fills.css` тАФ manual tokens not yet in Figma
6. `animations.css` тАФ shared keyframe library (`bds-spin`, `bds-pulse`, `bds-pop`, etc.) тАФ required by any component CSS that references these names

**Not bundled:** `bridge.css` (opt-in via separate export), `font-audit.css`
(Storybook-only), `motion-classes.css` (opt-in utility classes тАФ consumers import
directly if they want them), `storybook-themes.ts`, `index.ts`.

## Dark-mode selector contract

**Single switch across the ecosystem: `[data-theme="dark"]` attribute on `<html>`.**

- Figma-generated tokens: `:root[data-theme="dark"]`
- Brand overrides: `.theme-brand-brik[data-theme="dark"]`
- Storybook sets the attribute via the theme toolbar (`preview.tsx`)
- React consumers use `ThemeProvider` (which sets it automatically when `applyToBody` is true)
- Astro/Webflow/static consumers set it manually on `<html>`

No consumer should toggle a `.dark` class тАФ the attribute is the only switch.

## Decision tree тАФ where to put a change

| You want toтАж | File |
|---|---|
| Fix a value that came out of Figma wrong | `figma-dark-corrections.css` (dark) or a new `figma-corrections.css` (light тАФ doesn't exist yet, add when needed) |
| Add a semantic token Figma doesn't export | `gap-fills.css` |
| Adjust Brik's brand colors / fonts | `theme-brand-brik.css` тАФ consumers get these automatically when they apply `.theme-brand-brik` to `<body>` |
| Add a Webflow-facing alias | `bridge.css` (deprecated layer) |
| Touch an auto-generated file | Don't. Fix in Figma and re-pull, or add an override in a manual file. |

## Auto-generated тАФ do not edit

- `figma-tokens.css` тАФ regenerated by `scripts/figma-pull-tokens.sh` and Style Dictionary
- `figma-tokens-dark.css` тАФ same pipeline, dark-mode source

Any manual edit to these is overwritten on the next Figma sync. The correct
place for a correction is a manual file bundled AFTER the auto-generated ones.

## Manual (safe to edit)

- `figma-dark-corrections.css` тАФ known-wrong dark values
- `theme-brand-brik.css` тАФ Brik brand overrides (class-scoped, bundled into dist)
- `gap-fills.css` тАФ tokens Figma doesn't export yet
- `bridge.css` тАФ legacy Webflow aliases (deprecated, opt-in only)
- `font-audit.css`, `animations.css`, `motion-classes.css` тАФ Storybook/component-level

## Adding a new file to the bundle

1. Add the file to `tokens/`
2. Wire it into `scripts/build-dist-tokens.js` at the correct position (manual files after `figma-tokens-dark.css`, before `gap-fills.css` unless the semantic intent is different)
3. Run `npm run build:dist-tokens` and verify `dist/tokens.css` contains the expected declarations
4. Document here
