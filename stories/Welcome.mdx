# Brik Design System

BDS provides themed React components powered by CSS custom properties. All components adapt automatically across 9 bundled themes — switch using the paintbrush icon in the toolbar above.

Alongside the visual system, the **Brik Content System (BCS)** lives in this repo as the content-vocabulary peer: industry packs, voice patterns, and locked enums that pair with the visual tokens to power the portal's design + copy resolver.

## Token system

Components use CSS custom properties from the Style Dictionary token pipeline:

```
--[category]-[type]-[variant]
```

**Primitives** — raw scale values, never used directly in components:

- `--space-400` — 16px
- `--font-size-700` — 32px
- `--color-grayscale-darkest` — #333

**Semantic** — purpose-bound aliases, always use these in components:

- `--background-brand-primary` — brand background color
- `--font-family-heading` — heading font family
- `--padding-lg` — large spacing value
- `--border-radius-md` — button/card corner radius

## Toolbar reference

| Control | What it does |
|---------|--------------|
| Paintbrush | Switch between the 9 built-in BDS themes. Every component and doc page updates in real time. |
| Base Font | Set the root font size (14-20px). Scales all rem-based spacing and type tokens. |
| Animations | Toggle CSS animations and transitions on/off across all stories. |
| Font Audit | QA diagnostic theme. Assigns Georgia to heading, Verdana to body, Courier New to label. Exposes `--font-family-*` token misuse before a client theme does. |
