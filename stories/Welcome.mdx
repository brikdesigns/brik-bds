# Brik Design System — Storybook

This Storybook is the **live playground** for BDS components — every story renders the actual component with controls, ARIA, and theme switching. It's the canonical surface for:

- Live component playground (every `*.stories.tsx`)
- Visual regression (Chromatic)
- Component MCP endpoint (consumer agents query `/mcp` against `https://main--69b8918cac3056b39424d5d3.chromatic.com/`)
- Dev tools (Theme Switcher, Health Dashboard, Contrast Compliance, Token Coverage)

## What still lives here

- **Components.** Every `Components/*` story in the sidebar — Button, Card, Sheet, Form, etc. Click into any of them for the live playground.
- **Theme Switcher.** Use the paintbrush icon in the toolbar above to switch between Brik (light), Brik Dark, and Client Sim. The Client Sim theme assigns Georgia / Verdana / Courier New to heading / body / label families to expose font-family token misuse before a client theme reveals it.
- **Theme Switcher inspector** — open the [Theme Switcher](?path=/docs/theming-theme-switcher--docs) page for per-token inspection.
- **Contrast Compliance** — WCAG AA pair probe across the three built-in themes. Open via [Contrast Compliance](?path=/docs/overview-health-contrast-compliance--docs).
- **Health Dashboard** — coverage, audit signals.
- **Token Coverage** — utilization map across every token category.

## Architecture

BDS ships as a single package: `@brikdesigns/bds`. Consumers import:

```ts
import { Button, Card, Sheet, /* … */ } from '@brikdesigns/bds';
```

Plus the two CSS entry points:

```css
@import '@brikdesigns/bds/tokens.css';   /* full token cascade */
@import '@brikdesigns/bds/styles.css';   /* component CSS bundle */
```

For the full architecture (two-tier theming, four-layer composition, the LLM substrate role), see [design.brikdesigns.com](https://design.brikdesigns.com).

## Why this split?

Storybook is excellent at one job: rendering live, interactive components with controls. Trying to also be a long-form documentation site stretched it past its strengths — narrative pages, deep cross-linking, and search are what fumadocs at `design.brikdesigns.com` is built for. Each tool now does what it's good at.
