# Brik Design System

BDS is four tenets in one repository, published as a single versioned package. Each tenet is an independent vocabulary the portal, mockup generators, and client sites consume to ship work that looks and reads like Brik without anyone having to re-decide the basics every project.

Switch between built-in themes — Brik, Brik Dark, and Client Sim — using the paintbrush icon in the toolbar above, or open the [Theme Switcher](?path=/docs/theming-theme-switcher--docs) to inspect tokens per theme. Validate a theme before shipping via [Contrast Compliance](?path=/docs/overview-health-contrast-compliance--docs).

## The four tenets

| Tenet | Answers | Ships |
|---|---|---|
| **[Foundation](?path=/docs/foundations-design-tokens-overview--docs)** | *"What color, size, type?"* | Design tokens, scales, primitives |
| **[Theming](?path=/docs/theming-overview--docs)** | *"What brand, mood, shape?"* | Tokens + atmospheres + navigation archetypes + blueprints |
| **[Motion](?path=/docs/motion-overview--docs)** | *"How does it feel to move?"* | Three-tier animation system — CSS reveals → GSAP scroll → premium effects |
| **[Content](?path=/docs/content-overview--docs)** | *"What does it say, for whom?"* | Industry packs, voice patterns, locked vocabularies |

Each tenet evolves on its own cadence but intentionally cross-links at the edges — e.g. an industry pack declares a default atmosphere and navigation archetype, so the Content tenet drives defaults for the Theming tenet that the client brand then optionally overrides.

## Theming — the multi-tenant instrument

Theming composes four orthogonal layers. Each is a decision a client can override independently of the rest.

| Layer | Decides | Where it lives |
|---|---|---|
| [Tokens](?path=/docs/theming-layers-client-themes--docs) | Color, type, spacing values per brand | `theme-{client}.css` in the consumer repo |
| [Atmospheres](?path=/docs/theming-layers-atmospheres--docs) | Ambient mood — grain, glow, vignettes | CSS overlay from `@brikdesigns/bds/atmospheres/{slug}.css` |
| [Navigation Archetypes](?path=/docs/theming-layers-navigation-archetypes--docs) | Site-header shape + scroll/drawer behavior | Industry pack → BDS `<SiteHeader>` render |
| [Blueprints](?path=/docs/theming-layers-blueprints--docs) | Section composition + mood tagging | Authored in BDS, tagged per industry + mood |

Full architecture: [Theming Overview](?path=/docs/theming-overview--docs).

## Token system — the naming contract

Components use CSS custom properties from the Style Dictionary token pipeline:

```
--[category]-[type]-[variant]
```

**Primitives** — raw scale values, never used directly in components:

- `--space-400` — 16px
- `--font-size-700` — 32px
- `--color-grayscale-darkest` — #333

**Semantic** — purpose-bound aliases, always use these in components:

- `--background-brand-primary` — brand background color
- `--font-family-heading` — heading font family
- `--padding-lg` — large spacing value
- `--border-radius-md` — button/card corner radius

## Toolbar reference

| Control | What it does |
|---------|--------------|
| Paintbrush | Switch between the 3 built-in themes — Brik, Brik Dark, Client Sim. Every component and doc page updates in real time. |
| Base Font | Set the root font size (14-20px). Scales all rem-based spacing and type tokens. |
| Animations | Toggle CSS animations and transitions on/off across all stories. |
| Font Audit | QA diagnostic theme. Assigns Georgia to heading, Verdana to body, Courier New to label. Exposes `--font-family-*` token misuse before a client theme does. |
