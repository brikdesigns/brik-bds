# Size

Size tokens define dimensional constraints for containers, elements, and breakpoints.

## Size scale (primitives)

<SpacingScale scale={sizeScale} prefix="--size" />

## Semantic size tokens

### Container sizes

| Token | CSS Variable | Value |
|-------|-------------|-------|
| Full width | `--_size---container-size--full-width` | 100% |
| Three-quarter | `--_size---container-size--three-quarter-width` | 75% |
| Two-third | `--_size---container-size--two-third-width` | 66% |
| Half | `--_size---container-size--half-width` | 50% |
| One-third | `--_size---container-size--one-third-width` | 33% |
| Quarter | `--_size---container-size--quarter-width` | 25% |

### Screen breakpoints

| Token | CSS Variable | Value |
|-------|-------------|-------|
| Wider | `--_size---screen-size--wider` | 1440px |
| Wide | `--_size---screen-size--wide` | 1280px |
| Desktop | `--_size---screen-size--desktop` | 1080px |
| Tablet | `--_size---screen-size--tablet` | 768px |
| Mobile | `--_size---screen-size--mobile` | 320px |

### Hero constraints

| Token | CSS Variable | Value |
|-------|-------------|-------|
| Max hero height | `--_size---container-size--max-hero-height` | 100vh |
| Max hero width | `--_size---container-size--max-hero-width` | 100vw |

### Element sizes

Semantic size tokens follow the same tiny-to-huge scale as spacing:

| Token | Description |
|-------|-------------|
| `--_size---tiny` | 2px |
| `--_size---sm` | 4px |
| `--_size---md` | 8px |
| `--_size---lg` | 16px |
| `--_size---xl` | 24px |
| `--_size---xxl` | 32px |
| `--_size---huge` | 40px |
