# Motion

Motion tokens define the durations, easings, and keyframes shared across every animation in BDS. Consumers compose these via utility classes (<code>motion-classes.css</code>) or reference them directly in component CSS. For the system-level tier model, effect catalog, and vocabulary enums, see [Motion System Overview](?path=/docs/foundations-motion-overview--docs).

## Durations

BDS provides three semantic duration aliases — fast, normal, slow — that map to Figma-sourced primitive values. Component CSS always references semantic tokens, never the numeric primitives directly.

<DurationTable />

**Client theming:** Override any of these in a client's <code>theme-{'{client}'}.css</code> to change the animation personality without touching component code. A luxury brand might set <code>--duration-normal: 300ms</code>; a snappy SaaS dashboard might set it to <code>150ms</code>.

## Easings

Four easing curves cover the full range of UI motion intent. Each maps to a named CSS custom property so component CSS stays readable and client overrides remain possible.

<EasingTable />

**Figma note:** <code>--ease-in</code> and <code>--ease-out</code> originate in Figma but export as <code>"String value"</code> due to a Figma cubic-bezier limitation. The gap-fills in <code>tokens/gap-fills.css</code> override those with the correct <code>cubic-bezier()</code> values. <code>--ease-spring</code> and <code>--ease-in-out</code> are defined entirely in gap-fills (marked <code>TODO: promote to Figma</code>).

## Keyframes

All named keyframes live in <code>tokens/animations.css</code> — one source of truth. Component CSS references keyframe names rather than defining its own. Click **Replay** to preview each one.

<KeyframeTable />

### Reduced motion

<code>tokens/animations.css</code> includes a global <code>@media (prefers-reduced-motion: reduce)</code> block that collapses all <code>animation-duration</code> and <code>transition-duration</code> values to <code>0.01ms</code>. This affects every element, including components using utility classes — no per-component override needed.

## Utility Classes

<code>tokens/motion-classes.css</code> exposes composable classes built from the token layer. Apply them directly in JSX — no custom CSS needed for common patterns. Requires <code>animations.css</code> imported first.

```jsx
<Dot status="error" className="bds-anim-pulse" />
<div className="bds-enter-pop">New content</div>
<ul>
  <li className="bds-enter-fade bds-stagger-1">Item one</li>
  <li className="bds-enter-fade bds-stagger-2">Item two</li>
  <li className="bds-enter-fade bds-stagger-3">Item three</li>
</ul>
```

## Figma Sync

Duration and easing tokens are also Figma Variables and export to <code>build/figma/swift/BDSMotion.swift</code> as Swift constants for UIKit and SwiftUI animation use.
